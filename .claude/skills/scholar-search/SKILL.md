# scholar-search — 다중 학술 DB 검색 → 위키 투입

`tools/scholar_search.py` 로 여러 공개 논문 DB(arXiv·Crossref·OpenAlex)를 한 번에 검색해
**메타데이터**(제목·저자·연도·DOI·오픈액세스 링크)를 모으고, 여러 DB에서 동시에 나온 논문일수록
높은 점수를 준다. 결과는 `raw/` 에 저장해 `/new-wiki-page` 로 바로 위키화한다.

## 절차

1. **검색 실행** — 사용자가 준 주제로:
   ```powershell
   py tools/scholar_search.py "<주제>" --limit 10 --save
   ```
   - `--save` 면 `raw/scholar_<주제>_<날짜>.md` 에 결과를 저장한다.
   - 특정 소스만: `--sources arxiv,openalex`
2. **결과 검토** — 출력된 목록에서 점수(교차검증 DB 수)·연도를 보고 사용자에게 상위 후보를 요약 보고한다.
   원하는 논문이 없으면 검색어를 바꿔 재실행.
3. **위키화(선택)** — 사용자가 특정 논문을 위키에 넣고 싶어 하면:
   - 저장된 `raw/scholar_*.md` 를 근거로 `/new-wiki-page` 절차를 따른다.
   - **주의**: 이 노트는 *메타데이터*다. 정확한 내용 페이지를 쓰려면 원문(오픈액세스 PDF 링크 등)을
     사용자가 `raw/` 에 받아두거나, 초록 수준에서만 요약하고 "초록 기반"임을 페이지에 명시한다.

## 원칙 / 한계

- **메타데이터·오픈액세스 링크만** 다룬다. 유료 논문 PDF를 우회 수집하지 않는다(저작권·ToS 준수).
- 네트워크가 필요하다. 각 DB는 공개 API(키 불필요)이며, 한 곳이 막혀도 나머지 결과는 반환된다.
- 출처 없는 단정 금지 — 위키 페이지로 옮길 때 DOI/링크를 `## 참고 / 출처` 에 남긴다. (RULES.md 2절)
